# LifeOS PRD

## 1. 目标与定位

LifeOS 是一款面向个人的本地优先生产力系统，用于替代 Todoist、TickTick、日历与学习计划表。核心价值是在离线、单用户、无需登录的条件下提供流畅的桌面体验，并以 SQLite 作为唯一数据源。

## 2. 核心原则

- Local First：数据存储在本地 SQLite，功能离线可用。
- 单用户：无需多租户，所有设计聚焦个人效率。
- 隐私安全：不依赖云同步，不做远程登录。
- 现代 UI：深色主题，兼顾 Linear、Notion、TickTick 的最佳交互模式。

## 3. 目标用户与使用场景

- 知识工作者、学生、自由职业者，需要在桌面端管理任务、日程、学习计划。
- 场景：每日规划、周回顾、番茄钟、目标追踪、学习进度复盘。

## 4. 功能范围

### 4.1 必须实现

- 任务管理（优先级、状态、截止日、标签、子任务、提醒）。
- 日历视图（日/周/月，拖拽调整时间）。
- 学习计划表（学习目标、科目、计划时间段、进度）。
- 番茄钟（计时、日志、与任务/学习关联）。
- 收藏夹/备忘（轻量笔记，链接到任务或学习计划）。
- 全局搜索与快捷键。
- 深色主题（支持未来扩展浅色主题）。

### 4.2 可迭代

- 数据导出/导入。
- 仪表盘统计。
- 通知提醒（本地）。
- 多语言。

## 5. 用户故事

1. 作为用户，我要快速新建任务并设定优先级与截止日，以便管理工作。
2. 作为用户，我要在日历中拖拽任务，以便规划时间块。
3. 作为用户，我要创建学习计划，设定每周学习目标并追踪完成度。
4. 作为用户，我要使用番茄钟，自动记录专注时长并与任务关联。
5. 作为用户，我要在深色界面中通过全局搜索快速找到任务或笔记。

## 6. 非功能需求

- 性能：页面切换 <200ms，任务列表滚动 60fps。
- 数据：SQLite 本地存储，支持 WAL 模式。
- 可靠性：异常捕获与本地备份提示。
- 安全：本地数据不加密（可后续迭代），但遵循最小权限。

## 7. 设计约束

- 技术栈：Tauri + React + TypeScript + SQLite。
- 离线优先：所有功能必须在断网下运行。
- UI 风格：参照 Linear、Notion、TickTick 的简洁、功能密度与动效。

## 8. 成功指标

- 可完成任务、日历、学习计划的核心闭环。
- UI 达到现代桌面应用水准。
- 离线运行无阻断。

## 9. 版本规划

- MVP：任务、日历、学习计划、基础番茄钟。
- V1：全局搜索、统计、导入导出。
- V2：通知提醒、多语言、插件化。

## 10. 风险与开放问题

- 性能：大数据量下列表渲染需虚拟滚动。
- 数据迁移：未来如何支持导入现有 Todoist/TickTick 数据。
- 提醒：Tauri 本地通知能力需验证。
